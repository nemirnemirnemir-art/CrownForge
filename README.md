# CrownForge

**Roguelike-стратегия о защите королевства, строительстве городов и управлении героями.**

---

## О проекте

CrownForge — это амбициозная roguelike-игра в жанре castle-defense с элементами градостроительства и тактического менеджмента отрядов. Игрок выступает в роли правителя осаждённого королевства, которому предстоит одновременно:

- **Оборонять замок** от волн врагов в реальном времени
- **Строить и улучшать здания** на карте королевства
- **Нанимать героев и управлять отрядами** на поле боя
- **Управлять экономикой** — 13 типов ресурсов плюс золото (Денарии)
- **Применять заклинания и способности короля** в критические моменты

Игровой цикл: **Подготовка (стройка/найм/крафт) → Битва (защита от волн) → Награда (улучшения/артефакты) → Повтор с нарастающей сложностью.** Замок — центральная точка обороны: если он падёт, забег окончен.

**Движок:** Godot 4.3 (GL Compatibility) | **Язык:** GDScript | **Визуальный стиль:** 2D Pixel Art

## Архитектура и масштаб

Проект построен по принципу **Orchestrator + Module Decomposition**:

- **30 автозагрузок (Autoload)** — глобальные фасады, делегирующие логику узкоспециализированным модулям-помощникам
- **EventBus** — центральная шина событий для развязанной коммуникации между системами
- **Сцена-first композиция** — `.tscn` файлы как основной механизм компоновки
- **Ресурсные данные** — юниты, здания, заклинания, перки и улучшения определены как Godot Resources (`.tres`)

## Статус разработки

**Игра находится в активной стадии разработки.** При этом практически все ключевые механики уже реализованы и взаимодействуют между собой:

- Работает полный цикл: создание персонажа → строительство → волны врагов → награды → повтор
- Экономическая система функционирует: ресурсы добываются, тратятся на строительство и найм героев
- Боевая система обрабатывает героев, врагов, заклинания и способности короля в реальном времени
- Система улучшений зданий проверена через автоматический аудит: **111 тестов пройдено, 0 провалено**

### Инструменты отладки и тестирования

Для всестороннего тестирования механик разработано **мощное дебаг-меню** (клавиша `F10`), включающее:

- **Спавн-меню** — создание врагов, героев, заклинаний, предметов, ресурсов и артефактов по категориям
- **Панель аудита улучшений зданий** (`F11`) — живая матрица из 141 записи с отчётами в JSON/Markdown
- **Панель тестирования перков** — проверка эффектов каждого перка
- **FPS-оверлей** — счётчик кадров
- **323 автономных тестовых скрипта** в `scripts/dev/tests/`
- **Отдельные тестовые сцены**: `CombatTest` (боевая арена), `TenKingsPrototype` (карточная стратегия)
- **Горячие клавиши** для быстрого спавна (`Q`, `R`, `T`)
- **Headless QA-раннер** для автоматической валидации улучшений

## Запуск

1. Установите [Godot Engine 4.3](https://godotengine.org/)
2. Откройте проект в Godot (укажите папку `CrownForge` как корень проекта)
3. Нажмите **F5** для запуска игры или **F6** для запуска текущей сцены
