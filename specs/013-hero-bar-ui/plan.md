# Технический план: Панель героев (Hero Bar + Hero Card UI)

**Версия**: 1.0  
**Дата**: 2025-01-XX  
**Основано на**: `spec.md`  
**Feature Branch**: `013-hero-bar-ui`

---

## Обзор

Создание UI панелей для управления героями: нижняя панель героев (Hero Bar) и карточка героя справа (Hero Card). Расширение HeroCore для поддержки XP, смерти героев и управления отрядом.

**Цель**: Создать простой, казуальный UI для просмотра героев, выбора отряда (до 6 героев) и просмотра детальной информации о герое.

---

## Архитектура

### Компоненты

1. **HeroCore.gd** (расширение существующего автолоада)
   - Добавление полей: `xp`, `xp_to_next`, `is_dead`, `is_removed`
   - Новые методы: `set_hero_active()`, `is_hero_active()`, `add_xp_to_hero()`, `set_hero_dead()`, `remove_hero()`
   - Хранилище: `active_hero_ids: Array[String]` (максимум 6)
   - Авто-левелап при получении XP

2. **HeroBar.gd** (новый контроллер)
   - Управление нижней панелью с 2 рядами по 10 слотов (20 слотов)
   - Отображение фейсов героев, красных ободков для выбранных, черепов для мёртвых
   - Обработка кликов для выбора/снятия выбора героев
   - Поддержка пагинации (задел в архитектуре)

3. **HeroCard.gd** (новый контроллер)
   - Отображение карточки героя справа внизу
   - Показ имени, статов (HP, Damage, Level, XP), анимации walk
   - 4 кнопки-заглушки снизу

4. **HeroBar.tscn** (новая сцена)
   - Control узел с GridContainer (2 ряда × 10 колонок)
   - 20 слотов (Button или TextureRect) размером 50×50
   - Задел для кнопок пагинации (опционально)

5. **HeroCard.tscn** (новая сцена)
   - Control узел, закреплённый справа внизу
   - Композиция: окно героя (слева), имя (сверху справа), статы (посередине справа), кнопки (снизу)

6. **Интеграция с GameManager**
   - Сохранение/загрузка `active_hero_ids`, `xp`, `is_dead`, `is_removed` для каждого героя

**Цель**: Добавить поддержку XP, смерти героев и управления отрядом в HeroCore.

  - `is_dead: bool = false` — флаг смерти
  - `is_removed: bool = false` — флаг удаления (для освобождения слота)

- [ ] **T002** Добавить переменную `active_hero_ids: Array[String] = []` в HeroCore для хранения состава отряда

- [ ] **T003** Реализовать метод `set_hero_active(hero_id: String, active: bool) -> bool`:
  - Проверка, что герой существует и не мёртв (если active == true)
  - Если active == true и уже выбрано 6 героев — вернуть false
  - Если active == true — добавить hero_id в active_hero_ids (если ещё нет)
  - Если active == false — убрать hero_id из active_hero_ids
  - Вернуть true при успехе

- [ ] **T004** Реализовать метод `is_hero_active(hero_id: String) -> bool`:
  - Проверить, есть ли hero_id в active_hero_ids

- [ ] **T005** Реализовать метод `get_active_heroes() -> Array[Dictionary]`:
  - Вернуть массив героев из heroes, чьи ID есть в active_hero_ids

- [ ] **T006** Реализовать метод `add_xp_to_hero(hero_id: String, xp_amount: int) -> void`:
  - Найти героя по ID
  - Увеличить xp на xp_amount
  - Если xp >= xp_to_next — вызвать авто-левелап:
    - Увеличить level на 1
    - Случайно увеличить либо hp += 1, либо damage += 1
    - Обновить xp: xp = xp - xp_to_next (или обнулить, если xp < xp_to_next * 2)
    - Пересчитать xp_to_next для нового уровня (пока фиксируем 5 для всех уровней)
    - Вызвать recalcStats()

- [ ] **T007** Реализовать метод `set_hero_dead(hero_id: String, is_dead: bool) -> void`:
  - Найти героя по ID
  - Установить is_dead = is_dead
  - Если is_dead == true — автоматически убрать героя из active_hero_ids

- [ ] **T008** Реализовать метод `remove_hero(hero_id: String) -> void`:
  - Найти героя по ID
  - Установить is_removed = true
  - Убрать героя из active_hero_ids (если был выбран)
  - Опционально: можно удалить героя из heroes (но лучше оставить с флагом для истории)

- [ ] **T009** Обновить `createHero()` для инициализации новых полей:
  - `xp: 0`
  - `xp_to_next: 5`

- [ ] **T010** Обновить `getAllHeroes()` и `getUnlockedHeroes()` для фильтрации по `is_removed`:

## Технические детали

### Порядок отображения героев

Герои отображаются в порядке их ID (hero_1, hero_2, ...). Это соответствует порядку создания.

### Пагинация

В этом спеке реализуется только задел в архитектуре:
- Переменная `current_page` в HeroBar
- Вычисление `hero_index = current_page * 20 + slot_index`
- UI кнопок переключения страниц не реализуется (можно добавить позже)

### Формула XP

Для этого спека фиксируем простую формулу:
- Переход с уровня 1 на 2: требуется 5 XP
- Для всех уровней пока используем xp_to_next = 5 (можно расширить позже)

### Удаление героя

При освобождении слота (клик по черепу) герой помечается как `is_removed = true`. Он остаётся в списке heroes, но не отображается в UI. Это позволяет сохранить историю героев.

### Синхронизация UI

UI обновляется через:
1. Периодический вызов `update_display()` в `_process()` (простой вариант)
2. Сигналы от HeroCore при изменении состояния (более эффективный вариант, можно добавить позже)
