## 1. Термины и текущее состояние

### Термины

**Мясо** — общий ресурс еды. Должен быть один и тот же:
- в UI города,
- в UI основного боя,
- в UI полей охоты (а сейчас там отдельный "локальный" счётчик — это как раз проблема).

**Охотничьи палатки** (в Town) — здание, куда ты назначаешь рабочих/охотников. От него идёт пассивный прирост мяса в минуту, пока игрок не на сцене полей.

**Поля охоты** (сцена Hunting) — визуальная сцена:
- охотники бегают, охотятся;
- рабочие таскают мясо до палатки на самой сцене;
- сейчас добыча прибавляется только в локальный UI сцены, а не в общий ресурс.

### Текущая проблема

- UI полей охоты имеет отдельный локальный счётчик мяса
- Пассивный прирост от палаток не останавливается при входе на поля
- Доставки мяса на сцене охоты не идут в глобальный ресурс

---

## 2. Правило пассивной добычи (когда игрок НЕ на полях)

В Town у здания "Охотничьи палатки" есть:
- число назначенных рабочих/охотников `workers_count` (или `hunters_count`).

Пока игрок не на сцене полей:
- каждый рабочий даёт 2.2 мяса в минуту (текущее дефолтное число).
- формула примерно такая: `meat_passive_per_min = workers_count * 2.2`
- если `workers_count == 0`, то `meat_passive_per_min = 0` (нет рабочих = нет производства)

(точные цифры и "8 рабочих → 2.2×N" ты потом добьёшь тестами — для логики это не критично).

Этот пассив:
- идёт сразу в общий счётчик мяса,
- общий счётчик отражается и в городе, и в бою.

---

## 3. Что должно происходить при входе на поля охоты

Когда игрок заходит на сцену Hunting:

1. **Остановить пассив с палаток:**
   - больше не считаем `workers_count * 2.2 / мин` до тех пор, пока игрок на полях.

2. **Зафиксировать мясо на входе:**
   - берем текущее значение общего мяса (после всех пассивов)
   - это число показывается:
     - в UI полей,
     - и, по сути, это же значение есть в городе и бою (один ресурс).

3. **С этого момента:**
   - мясо больше не капает "из воздуха" по 2.2/мин,
   - оно добавляется только тогда, когда рабочие на сцене физически доносят мясо до палатки.

**Важно:**
- сцена полей не должна держать свой отдельный ресурс мяса,
- все "доставки" должны бить по общему ресурсу (см. следующий пункт).

---

## 4. Что должно происходить на сцене охоты (онлайн-добыча)

Сейчас уже сделано:
- рабочие бегают,
- собирают мясо,
- носят его до палатки,
- UI сцены обновляется (типа "+мясо").

### Что нужно изменить:

В момент, когда рабочий донёс мясо до палатки:
- не увеличивать локальную переменную сцены,
- а вызывать что-то вроде `GameState.add_meat(amount)` (или твой глобальный сервис ресурса).

**UI:**
- UI сцены полей должен показывать тот же самый ресурс, что и:
  - UI города,
  - UI боя.

То есть все три UI должны читать одно значение:
- например, `GameState.meat` или `Resources.meat`.

### Поведение:

**До входа на поля:**
- мясо росло за счёт 2.2/мин/рабочий (пассив палаток).

**В момент входа:**
- мы просто видим текущее накопленное значение.

**Пока сидим на полях:**
- пассив заморожен,
- мясо идёт только от рабочих, которые физически бегают и приносят его.

**При выходе:**
- снова включаем пассив палаток.

## 5. Связка с городом и основным боем

### Итоговая цель по связке:

Один глобальный счётчик мяса, лежит где-то в автолоаде/сервисе.

**Источники этого счётчика:**
- Пассив с палаток (Town) — работает только пока игрок не на полях.
- Реальные доставки мяса на сцене (Hunting) — работают только пока игрок на полях.

**UI:**
- Город: показывает `meat`.
- Бой: показывает `meat`.
- Поля охоты: показывают `meat`.
- TODO (опционально, не в MVP): можно отдельной цифрой вывести +X за эту сессию на полях.

## 6. Краткое резюме изменений

1. **Пассивный прирост:**
   - считать и добавлять в глобальное мясо, но выключать его на время пребывания на сцене охоты.

2. **Сцену охоты:**
   - переподключить, чтобы все "донесения" мяса рабочими шли в глобальный ресурс, а не в локальный счётчик UI.

3. **Все UI:**
   - должны читать одну и ту же переменную мяса, а не каждый свой локальный state.

## 7. Архитектурные изменения

### 7.1. Глобальный ресурс мяса

**Текущее состояние:**
- Мясо хранится в `TownCore._food` (float)
- `HuntingCore.deliver_meat()` вызывает `EconomyCore.add_food()`, который перенаправляет в `TownCore.add_food()`

**Требуется:**
- Убедиться, что все UI читают `TownCore.get_food()`
- Добавить механизм остановки пассивного прироста при входе на сцену охоты

### 7.2. Пассивный прирост от палаток

**Текущее состояние:**
- В `HuntingCore` есть константа `MEAT_PER_HUNTER_PER_MINUTE: float = 2.2`
- Офлайн-прогресс рассчитывается в `_calculate_offline_earnings()`

**Требуется:**
- Создать отдельный модуль `MeatProductionCore` (или `ResourceProductionCore`) для управления всеми источниками пассивной добычи мяса
- Модуль отвечает только за расчёт "сколько и откуда капает" и вызывает `TownCore.add_food(delta)`
- Хранение самого ресурса мяса остаётся в `TownCore._food`
- Модуль должен останавливать пассивный прирост при входе на сцену охоты
- Модуль должен возобновлять пассивный прирост при выходе со сцены охоты

### 7.3. Сцена охоты

**Текущее состояние:**
- `HuntingCore.deliver_meat()` уже вызывает `EconomyCore.add_food()`
- UI сцены может иметь локальный счётчик

**Требуется:**
- Убедиться, что UI сцены читает глобальный ресурс
- Убрать локальные переменные мяса из UI сцены

## 8. API изменения

### MeatProductionCore (новый модуль)
- Создать автолоад `MeatProductionCore` для управления пассивной добычей мяса
- Методы:
  - `set_hunting_passive_enabled(enabled: bool)` — включить/выключить пассив охоты
  - `get_passive_meat_per_minute() -> float` — получить текущий пассивный прирост (возвращает 0, если нет рабочих или пассив выключен)
  - `_process(delta)` — в цикле рассчитывать и вызывать `TownCore.add_food(delta * meat_per_sec)` только если пассив включён и есть рабочие
- Получает количество рабочих из `TownCore.get_building_workers("hunting_tents")`
- Если `workers_count == 0`, пассивный прирост = 0 (нет рабочих = нет производства)
- Сохраняет состояние включённости пассива в сейв (для восстановления при загрузке)

### TownCore
- Хранит сам ресурс мяса (`_food`)
- Предоставляет API для чтения: `get_food() -> float`
- Предоставляет API для изменения: `add_food(amount: float)`
- Не управляет пассивным приростом (это делает `MeatProductionCore`)

### HuntingCore
- При входе на сцену (в `_ready()` или при инициализации сцены):
  - Вызывать `on_enter_scene()`:
    - Установить `hunting_state.is_on_hunting_scene = true`
    - Обновить `hunting_state.last_seen_timestamp`
    - Вызвать `MeatProductionCore.set_hunting_passive_enabled(false)`
    - Вызвать `HuntingSceneTheatre.restore_snapshot(dt)` с вычисленным `dt`
- При выходе со сцены (в `_exit_tree()` или при деинициализации сцены):
  - Вызывать `on_exit_scene()`:
    - Установить `hunting_state.is_on_hunting_scene = false`
    - Обновить `hunting_state.last_seen_timestamp`
    - Вызвать `HuntingSceneTheatre.save_snapshot()`
    - Вызвать `MeatProductionCore.set_hunting_passive_enabled(true)`
- `deliver_meat()` уже корректно добавляет в глобальный ресурс через `EconomyCore.add_food()`
- Использовать сигналы/события для явного управления состоянием (не проверять текущую сцену в _process)
- Управлять сохранением/загрузкой `hunting_state` через `SaveCore`

### UI компоненты
- Все UI должны читать `TownCore.get_food()` вместо локальных переменных

### HuntingSceneTheatre (новый компонент)
- Отвечает за снапшот и восстановление "театра" (позиции/состояния юнитов при входе/выходе/загрузке)
- Может быть частью `HuntingCore` или отдельным модулем
- Методы:
  - `save_snapshot() -> Dictionary` — сохранить текущее состояние всех юнитов
  - `restore_snapshot(dt_seconds: float) -> void` — восстановить состояние с учётом прошедшего времени
- **Важно:** Театр никогда не начисляет мясо — это чистый визуал. Экономика работает отдельно через реальные доставки.

## 8.1. Система сохранения состояния сцены охоты

### Хранимое состояние (для сейва)

В структуру сохранения игры добавляется блок:

```gdscript
hunting_state:
  is_on_hunting_scene: bool          # игрок в момент сейва был на полях или нет
  last_seen_timestamp: int           # реальное время (unix time, сек) последнего "наблюдения" полей
  snapshot_units: Array[Dictionary]  # сериализованный снапшот юнитов (если игрок был на полях)
    - id: string                     # уникальный id юнита
    - kind: "sheep" | "hunter" | "worker"
    - pos: Vector2
    - velocity: Vector2
    - state: "IDLE" | "WALK" | "PATROL" | "CHASE" | "ATTACK" | ...
    - target_id: string | null       # для охотника
    - anim_phase: float              # фаза анимации (для рассинхрона)
```

**Примечание:** Если `is_on_hunting_scene == false`, `snapshot_units` можно либо не сохранять, либо сохранять как "последний видимый кадр" для театра (опционально).

### Смена сцены: вход/выход с полей

#### Сигналы

Через EventBus (или напрямую) гарантируем два события:
- `HuntingCore.on_enter_scene()` — вызывается при `_ready()` сцены охоты
- `HuntingCore.on_exit_scene()` — вызывается при `_exit_tree()` сцены охоты

#### on_enter_scene()

1. Пометить состояние:
   - `hunting_state.is_on_hunting_scene = true`
   - `hunting_state.last_seen_timestamp = Time.get_unix_time_from_system()`

2. Сообщить `MeatProductionCore`, что пассив охоты выключен:
   - `MeatProductionCore.set_hunting_passive_enabled(false)`

3. Дожать оффлайн-добычу города в целом (если есть общий оффлайн), но **не добавлять** за это время пассив охоты — она была выключена, пока игрок "внутри" полей.

4. Восстановить "театр" из `snapshot_units` и `last_seen_timestamp`:
   - Вычислить `dt = max(0.0, float(now - last_seen_timestamp))`
   - Вызвать `HuntingSceneTheatre.restore_snapshot(dt)`

#### on_exit_scene()

1. Обновить `hunting_state`:
   - `hunting_state.is_on_hunting_scene = false`
   - `hunting_state.last_seen_timestamp = Time.get_unix_time_from_system()`

2. Сохранить свежий снапшот юнитов:
   - `HuntingSceneTheatre.save_snapshot()`

3. Включить пассив охоты:
   - `MeatProductionCore.set_hunting_passive_enabled(true)`

4. Дальше, пока игрок в городе/бою, пассив охоты даёт `2.2 * hunters_count` мяса в минуту.

### Поведение при сохранении

#### 1. Сейв, когда игрок на полях

В момент сохранения:
- `hunting_state.is_on_hunting_scene = true`
- `hunting_state.last_seen_timestamp = Time.get_unix_time_from_system()`
- `HuntingSceneTheatre.save_snapshot()`
- `TownCore` сохраняет текущее значение мяса
- Пассив охоты в этот момент выключен и в сейв попадает уже выключенным (через состояние `MeatProductionCore` или флаг)

#### 2. Сейв, когда игрок не на полях

- `hunting_state.is_on_hunting_scene = false`
- `last_seen_timestamp` можно обновить (последнее время выхода / последнего снапшота)
- `snapshot_units` опционален (если хочешь сохранять "последний кадр" для театра)

### Поведение при загрузке

#### 1. Игрок сохранялся на полях (is_on_hunting_scene == true)

При загрузке:

1. Поднимается глобальное состояние игры (ресурсы, здания, `MeatProductionCore` и т.д.)

2. Из `hunting_state` читаем:
   ```gdscript
   var was_on_hunting_scene = hunting_state.is_on_hunting_scene
   var last_seen = hunting_state.last_seen_timestamp
   var now = Time.get_unix_time_from_system()
   var dt = max(0.0, float(now - last_seen))
   ```

3. Игра загружает сцену охоты, а в `HuntingCore._ready()`:
   - Выключает пассив охоты: `MeatProductionCore.set_hunting_passive_enabled(false)`
   - Вызывает `HuntingSceneTheatre.restore_snapshot(dt)`

**Важно:** За время между сейвом и загрузкой мясо от охоты **не начисляется**, потому что игрок "находился на полях", пассив выключен, а театр по определению не влияет на экономику.

#### 2. Игрок сохранялся вне полей (is_on_hunting_scene == false)

1. Игра загружает либо город, либо бой — как по дефолту
2. `MeatProductionCore` включает пассив охоты
3. Если игрок потом перейдёт на поля:
   - Время для `dt` берётся от `last_seen_timestamp`
   - Используется та же логика театра

### HuntingSceneTheatre: восстановление "живого" кадра

#### Функция

```gdscript
func restore_snapshot(dt_seconds: float) -> void:
    # Восстановление состояния юнитов с учётом прошедшего времени
```

#### Диапазоны времени

**dt ≤ 5 секунд — замороженный кадр**

Восстанавливаем `snapshot_units` как есть:
- Те же позиции
- Те же состояния
- Те же цели

После этого обычный AI продолжает работать.

**Эффект:** "я только что здесь был"

**5 < dt ≤ 15 секунд — мягкий сдвиг**

Для каждого юнита считаем `local_dt = clamp(dt_seconds, 0.0, 8.0–10.0)`.

**Овцы:**
- Сдвинуть по сохранённому `velocity` (`pos += velocity * local_dt`) с проверкой границ
- С небольшим шансом переназначить цель "пожрать траву" недалеко

**Охотники:**
- Если `state == CHASE` — двигать ближе к цели (овце); если достаточно близко — перевести в `ATTACK`/позицию рядом
- Если `IDLE`/`PATROL` — немного продвинуть по маршруту (без телепорта)

Никаких прыжков через полкарты.

**Эффект:** "они чуть-чуть продолжили то, что делали"

**dt > 15 секунд — новая естественная сцена**

На основе `snapshot` можно:
- Часть овец/охотников оставить примерно там же с шумом
- Часть перегенерировать по шаблону:
  - Овцы раскиданы по полю, часть идёт, часть стоит
  - 1–2 охотника в `CHASE`/`ATTACK`, остальные патрулируют

**Важно:** Всё выглядит как "устоявшийся хаос", а не идеальный грид.

#### Общие правила театра

1. **Театр никогда не начисляет мясо** — это чистый визуал
2. Сохраняем для юнитов `anim_phase`, чтобы анимации не стартовали синхронно при входе
3. При любом входе/загрузке после `restore_snapshot(dt)` всё дальше делает обычный AI (овцы бегают, охотники охотятся, рабочие таскают)

## 9. Тестовые сценарии

1. **Пассивный прирост:**
   - Назначить рабочих в палатки
   - Убедиться, что мясо растёт в UI города
   - Убедиться, что мясо растёт в UI боя
   - Войти на поля охоты
   - Убедиться, что пассив остановился

2. **Активная добыча:**
   - Войти на поля охоты
   - Дождаться доставки мяса рабочим
   - Убедиться, что мясо добавилось в глобальный ресурс
   - Убедиться, что UI города и боя обновились

3. **Переключение между сценами:**
   - Находиться в городе/бою, мясо растёт пассивно
   - Войти на поля, пассив остановился
   - Выйти с полей, пассив возобновился

4. **Сохранение/загрузка на сцене охоты:**
   - Сохранить игру, находясь на полях
   - Загрузить сейв
   - Убедиться, что пассив охоты выключен
   - Убедиться, что юниты восстановлены из снапшота
   - Убедиться, что мясо не начислялось за время между сейвом и загрузкой

5. **Театр сцены (визуальное восстановление):**
   - Сохранить игру на полях
   - Загрузить через 3 секунды — замороженный кадр
   - Загрузить через 10 секунд — мягкий сдвиг
   - Загрузить через 30 секунд — новая естественная сцена
