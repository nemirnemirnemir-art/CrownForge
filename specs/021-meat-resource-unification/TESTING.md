# Тестирование: Унификация ресурса мяса

**Дата:** 2025-01-27  
**Фаза:** 7 - Testing & Validation

---

## Чеклист тестирования

### 1. Базовая функциональность

#### 1.1. Инициализация модулей
- [ ] При входе на поле охоты все модули инициализируются корректно
- [ ] Нет ошибок в консоли при загрузке сцены
- [ ] Все подсистемы создаются и добавляются в дерево сцены
- [ ] Связи между модулями установлены правильно

#### 1.2. Отображение глобального мяса
- [ ] UI поля охоты показывает глобальное количество мяса из `TownCore.get_food()`
- [ ] При изменении мяса в городе, UI поля охоты обновляется
- [ ] При доставке мяса на поле, UI обновляется сразу

#### 1.3. Спавн охотников
- [ ] Количество охотников соответствует количеству назначенных рабочих
- [ ] При изменении количества рабочих, охотники обновляются при следующем входе
- [ ] Охотники не дублируются при перезаходах
- [ ] Охотники правильно восстанавливаются из снапшота

#### 1.4. Спавн овец
- [ ] На поле всегда 6 овец (TARGET_SHEEP_COUNT)
- [ ] При смерти овцы, через 15 секунд появляется новая
- [ ] Овцы правильно восстанавливаются из снапшота
- [ ] Застрявшие овцы телепортируются обратно

---

### 2. Пассивное производство мяса

#### 2.1. Когда игрок НЕ на поле охоты
- [ ] Пассивное производство включено (`MeatProductionCore._hunting_passive_enabled == true`)
- [ ] Мясо начисляется: `2.2 * workers_count` мяса в минуту
- [ ] Если `workers_count == 0`, производство = 0
- [ ] Мясо добавляется в `TownCore.add_food()`

#### 2.2. Когда игрок НА поле охоты
- [ ] Пассивное производство выключено (`MeatProductionCore._hunting_passive_enabled == false`)
- [ ] Мясо НЕ начисляется пассивно
- [ ] Мясо начисляется только от реальных доставок рабочими

#### 2.3. Переключение между сценами
- [ ] При входе на поле: пассив выключается
- [ ] При выходе с поля: пассив включается
- [ ] При загрузке игры на поле: пассив остается выключенным

---

### 3. Сохранение и загрузка

#### 3.1. Сохранение состояния
- [ ] При выходе с поля сохраняется снапшот юнитов
- [ ] Сохраняются позиции, скорости, состояния охотников
- [ ] Сохраняются позиции, скорости, состояния овец
- [ ] Сохраняется `is_on_hunting_scene: true/false`
- [ ] Сохраняется `last_seen_timestamp`

#### 3.2. Загрузка состояния
- [ ] При загрузке игры на поле, состояние восстанавливается
- [ ] Охотники восстанавливаются в правильных позициях
- [ ] Овцы восстанавливаются в правильных позициях
- [ ] Пассивное производство остается выключенным

#### 3.3. Театр сцены (Scene Theatre)
- [ ] `dt <= 5s`: Замороженный кадр (позиции как при выходе)
- [ ] `5s < dt <= 15s`: Мягкий сдвиг (юниты немного продвинулись)
- [ ] `dt > 15s`: Новая естественная сцена (регенерация с шумом)
- [ ] Театр НЕ влияет на экономику (не начисляет мясо)

#### 3.4. Reset прогресса
- [ ] При `reset_progress()` состояние охоты очищается
- [ ] `hunting_state.snapshot_units` очищается
- [ ] При следующем входе на поле - чистая сцена

---

### 4. Интеграция модулей

#### 4.1. Связи между модулями
- [ ] `HuntingSceneEvents` → `HuntingSceneSpawner` (animal_killed)
- [ ] `HuntingSceneEvents` → `HunterField` (meat_delivered, food_changed)
- [ ] `HuntingScenePersistence` → `HuntingSceneSpawner` (alive_sheep)
- [ ] Все сигналы подключены и работают

#### 4.2. Доступ к данным
- [ ] `spawner.alive_sheep` доступен из `persistence`
- [ ] `spawner.TARGET_SHEEP_COUNT` доступен из `HunterField`
- [ ] Все публичные методы доступны

#### 4.3. Cleanup
- [ ] При выходе со сцены все таймеры останавливаются
- [ ] Все сигналы отключаются
- [ ] Нет утечек памяти

---

### 5. Граничные случаи

#### 5.1. Нулевые значения
- [ ] `workers_count == 0`: нет охотников на поле
- [ ] `workers_count == 0`: пассивное производство = 0
- [ ] Пустой снапшот: создается новая сцена

#### 5.2. Крайние значения
- [ ] `workers_count == 10+`: все охотники спавнятся
- [ ] Много овец (>6): лишние не спавнятся
- [ ] Очень большой `dt` (>1 час): естественная регенерация

#### 5.3. Ошибки
- [ ] Нет `HuntingCore`: ошибка обрабатывается корректно
- [ ] Нет `TownCore`: ошибка обрабатывается корректно
- [ ] Нет `camera`: используются дефолтные позиции спавна

---

### 6. Производительность

#### 6.1. Оптимизация
- [ ] Нет избыточного логирования в продакшене
- [ ] Таймеры не создают утечек
- [ ] Рекурсивный поиск овец не тормозит

#### 6.2. Память
- [ ] Нет накопления охотников при перезаходах
- [ ] Нет накопления овец при перезаходах
- [ ] Снапшоты не растут бесконечно

## Сценарии тестирования

### Сценарий 1: Первый вход на поле
1. Игрок никогда не был на поле охоты
2. Войти на поле охоты
3. **Ожидаемый результат:**
   - Создается новая сцена (6 овец, охотники по количеству рабочих)
   - Пассивное производство выключено
   - UI показывает глобальное мясо

### Сценарий 2: Переход туда-обратно
1. Игрок в городе (5 рабочих назначено)
2. Войти на поле охоты
3. Подождать 10 секунд
4. Выйти с поля
5. **Ожидаемый результат:**
   - Снапшот сохранен
   - Пассивное производство включено
   - При следующем входе: `dt ≈ 10s`, мягкий сдвиг

### Сценарий 3: Сохранение/загрузка на поле
1. Игрок на поле охоты
2. Сохранить игру
3. Закрыть игру
4. Загрузить игру
5. **Ожидаемый результат:**
   - Игрок на поле охоты
   - Юниты восстановлены
   - Пассивное производство выключено

### Сценарий 4: Reset прогресса
1. Игрок на поле охоты (100 охотников наспавнено)
2. Нажать "Reset Progress"
3. Войти на поле охоты
4. **Ожидаемый результат:**
   - Чистая сцена (нет старых охотников)
   - Только охотники по количеству рабочих

### Сценарий 5: Изменение количества рабочих
1. Игрок в городе (3 рабочих)
2. Войти на поле: должно быть 3 охотника
3. Выйти, назначить 7 рабочих
4. Войти на поле: должно быть 7 охотников
5. **Ожидаемый результат:**
   - Охотники обновляются корректно
   - Нет дублирования
