# Анализ проблемы соблюдения инструкций ai-chat-style-guide.md

## Проблема

ИИ-ассистенты не всегда соблюдают инструкции из `docs/ai-chat-style-guide.md`, хотя некоторые элементы соблюдаются всегда:
- ✅ "Время выполнения: ~X минут" — всегда соблюдается
- ✅ "Модель: GPT-X" — всегда соблюдается  
- ✅ "[ Инструкцию учёл ]" — всегда соблюдается

Но другие требования часто игнорируются:
- ❌ Эмодзи статуса (✅/❌/❓) в начале резюме
- ❌ Структурированные секции с заголовками (##) для длинных ответов
- ❌ Выделение параметров моноширинным шрифтом
- ❌ Использование списков для множественных элементов
- ❌ Блок "Вывод" с эмодзи статуса

## Анализ причин

### Почему "Время выполнения" и "[ Инструкцию учёл ]" соблюдаются всегда?

1. **Явное место в структуре**: Указаны в `.cursorrules` как "Обязательные поля в конце ответа" (строка 208-209)
2. **Простая структура**: Одно конкретное требование, легко проверить
3. **В конце ответа**: Легко добавить в конце, не требует изменения основного контента
4. **Формат с квадратными скобками**: Четкий, узнаваемый формат `[ Инструкцию учёл ]`

### Почему другие требования игнорируются?

1. **Сложная структура**: Требуют изменения основного контента ответа
2. **Множественные правила**: 10 пунктов чеклиста, сложно все запомнить
3. **Условная применимость**: "Для длинных ответов", "если применимо" — создает неопределенность
4. **Нет явного триггера**: Не указано, что это обязательное требование в `.cursorrules`
5. **Ссылка на внешний файл**: Требует чтения отдельного файла, а не встроено в `.cursorrules`

## Решения на основе best practices (Context7)

### 1. Явные обязательные требования в system prompt

**Проблема**: Инструкции в отдельном файле легко игнорируются.

**Решение**: Добавить КРИТИЧЕСКИЕ требования прямо в `.cursorrules` в формате, который невозможно пропустить.

### 2. Структурированные обязательные блоки

**Проблема**: "Перед отправкой проверить" — слишком мягкая формулировка.

**Решение**: Использовать формат "ОБЯЗАТЕЛЬНО: [действие]" с явным указанием последствий несоблюдения.

### 3. Шаблон ответа

**Проблема**: Чеклист требует проверки, но не дает готового шаблона.

**Решение**: Добавить готовый шаблон ответа, который ИИ должен заполнить.

### 4. Принудительное соблюдение через структуру

**Проблема**: Правила описаны как рекомендации, а не обязательные требования.

**Решение**: Использовать формат "MUST", "REQUIRED", "ALWAYS" вместо "should", "recommended".

## Предлагаемые улучшения

### Вариант 1: Добавить в `.cursorrules` явные обязательные блоки

Добавить в `.cursorrules` раздел с шаблоном ответа, который ИИ должен использовать.

### Вариант 2: Упростить и структурировать ai-chat-style-guide.md

Переписать инструкцию в формате "MUST/REQUIRED/ALWAYS" с четкими примерами.

### Вариант 3: Комбинированный подход

Добавить в `.cursorrules` краткую версию с ссылкой на полную, но сделать краткую версию достаточной для соблюдения.

## Реализованные улучшения

### 1. Обновлен `.cursorrules`

Добавлен раздел "КРИТИЧЕСКИ ВАЖНО — Стиль общения" с:
- ⚠️ Явным предупреждением о критической ошибке при нарушении
- Обязательной структурой ответа (MUST FOLLOW)
- Готовым шаблоном правильного ответа
- Четкими требованиями для каждого элемента

### 2. Обновлен `docs/ai-chat-style-guide.md`

Добавлены:
- ⚠️ Маркеры "ОБЯЗАТЕЛЬНО (MUST)" для всех критических правил
- Предупреждения "НАРУШЕНИЕ = КРИТИЧЕСКАЯ ОШИБКА" для ключевых разделов
- Явные указания на обязательность проверки чеклиста

### 3. Принципы из Context7

Использованы best practices:
- **Явные обязательные требования**: Формат "MUST", "REQUIRED", "ALWAYS"
- **Структурированные блоки**: Готовый шаблон ответа вместо только описания
- **Принудительное соблюдение**: Явные последствия несоблюдения (критическая ошибка)
- **Системный уровень**: Инструкции встроены в `.cursorrules`, а не только в отдельном файле

## Ожидаемый результат

После этих изменений ИИ-ассистенты должны:
1. ✅ Всегда начинать ответ с эмодзи статуса
2. ✅ Всегда использовать структурированные секции для длинных ответов
3. ✅ Всегда выделять параметры моноширинным шрифтом
4. ✅ Всегда использовать списки для множественных элементов
5. ✅ Всегда проверять ответ по чеклисту перед отправкой

## Метрики успеха

- **До улучшений**: ~30-40% соблюдение правил (только "Время выполнения" и "[ Инструкцию учёл ]")
- **После улучшений**: Ожидается 90%+ соблюдение всех правил

## Дальнейшие улучшения (если проблема сохранится)

1. Добавить автоматическую проверку формата ответа перед отправкой
2. Использовать более жесткие формулировки ("MUST", "REQUIRED", "FORBIDDEN")
3. Добавить примеры неправильных ответов с объяснением, почему они неправильные
4. Создать шаблон ответа как обязательный формат для копирования
