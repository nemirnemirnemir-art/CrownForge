# Weapon Size Unification

## Goal

Привести все 20 оружий к единому поведению по размерам:

- В 2D-виде в редакторе и в игре визуальный спрайт/анимация и коллизия совпадают по размеру и позиции.

- Все апгрейды по размеру (тома, перки и т.п.) увеличивают и спрайт, и коллизию:
  - строго на +5% от базового размера за уровень;
  - пропорционально, без рассинхрона.

- Исправить баги, когда:
  - в игре коллизия больше/меньше PNG/AnimationSprite2D;
  - при каждом запуске/томе коллизия "раздувается" до гигантских размеров;
  - разные оружия масштабируются по-разному.

## Constraints

- Не менять баланс урона/скорости/частоты выстрела.
- Не трогать базовый размер артов (PNG) — только масштаб и коллизии.
- Ничего не удалять из сцен, кроме явно лишнего/дублирующего кода масштабирования.

## Task 1 — Инвентаризация оружия

Найти все сцены/скрипты 20 оружий и зафиксировать (используя автоматический поиск по структуре проекта — сканирование папок с оружиями, поиск `.tscn` файлов с характерными узлами):

- какие узлы отвечают за визуал (Sprite2D, AnimatedSprite2D, AnimationPlayer/AnimatedSprite2D);
- какие узлы отвечают за коллизию (Area2D/CollisionShape2D/CollisionPolygon2D/RayCast2D и т.п.);
- где и как сейчас меняется размер (scale у узлов, изменение shape.size/radius, отдельные переменные size/scale и т.д.).

Составить краткую таблицу/коммент по каждому оружию:

`weapon_name: visual_node(s), collider_node(s), current_scale_logic (code refs).`

## Task 2 — Единая модель размера

Для каждого оружия ввести/принять единый контракт:

### Базовый размер:

- `base_scale` для визуального узла (Vector2, обычно (1,1));
- `base_collider_size` или `base_collider_scale` для коллизии:
  - либо как Vector2 / float (radius, length);
  - либо через `base_scale` того же узла, если коллизия сидит на дочернем узле и унаследует scale.

### Текущий размер рассчитывать только из базы + модификаторы, без накопления:

```
size_level      // int, уровень "size / тома", читается из конфига оружия (.tres)
SIZE_STEP = 0.05

base_scale_factor = 1.0 + size_level * SIZE_STEP
other_modifiers = 1.0  // множитель от пассивок героя, перков и т.п.
scale_factor = base_scale_factor * other_modifiers
```

### Применение:

**Визуал:**
```
visual_node.scale = base_scale * scale_factor
```

**Коллизия:**

если используется общий scale узла — аналогично:
```
collider_node.scale = base_collider_scale * scale_factor
```

если меняется размер shape:
```
shape.size = base_shape_size * scale_factor    // Rect/capsule
shape.radius = base_radius * scale_factor      // Circle
shape.height = base_height * scale_factor      // beam length
```

## Task 3 — Удалить баги с "раздуванием"

**Важно:** Эта задача выполняется ПЕРЕД Task 2 (применение новой системы) для снижения риска конфликтов и упрощения отладки.

Нужно найти и исправить места, где масштаб накапливается:

### Примеры проблемных паттернов (поискать и переписать):

- `node.scale *= 1.05` при каждом апгрейде / каждом запуске;
- `shape.size *= 1.05` или `shape.radius *= 1.05` многократно;
- повторное применение модификатора при каждом `_ready()` или при respawn.

### Все такие места заменить на вычисление от базовых значений, например:

```gdscript
# вместо этого
func apply_size_upgrade():
    sprite.scale *= 1.05

# должно быть
func apply_size_upgrade():
    var scale_factor = 1.0 + size_level * 0.05
    sprite.scale = base_scale * scale_factor
```

**Важно:** в `_ready()` и при инстансе оружия всегда сначала восстанавливать базовые значения (`base_*`), потом применять модификаторы.

## Task 4 — Синхронизация 2D редактора и рантайма

Для каждого оружия:

### В редакторе (2D-вью) выставить:

- `base_scale` спрайта/анимации = то, как оружие должно выглядеть на уровне 0;
- коллайдер так, чтобы он накрывал видимый спрайт логично (не меньше/не больше).

### Убедиться, что в коде:

- в `_ready()` не происходит лишнего масштабирования, которое ломает то, что видишь в редакторе;
- если нужен стартовый бонус размера (например, от пассивок героя) — он считается через `scale_factor`, а не через ручной `*=`, и не затирает базу.

## Task 5 — Тест-кейсы

Для каждого оружия сделать минимальный набор проверок (можно через Debug-сцену):

### Base level (size_level = 0):

- визуал и коллизия совпадают;
- при нескольких рестартах/спавнах подряд размер не растёт и не уменьшается.

### Несколько уровней size/tome:

- при поднимании размера с уровня 0 → 1 → 2 → …:
  - спрайт и коллизия увеличиваются одинаково;
  - не появляется "прыжков" (например, коллизия вдруг в 2 раза больше);
  - при достижении 10 уровней размер ≈ 1.5x от базы (10 × 5%).

### Запуск сцены несколько раз:

- никакого накопительного роста: 10 запусков подряд = тот же размер при одном и том же `size_level`.
