## Цель упражнения

Научиться применять hue shift (сдвиг оттенка) для создания объема и реалистичного освещения в пиксель-арте. Вместо простого затемнения/осветления цветов, hue shift создает более естественный и приятный визуальный эффект.

**Что вы освоите**:
- Применение hue shift для теней (сдвиг в сторону синего/фиолетового)
- Применение hue shift для бликов (сдвиг в сторону желтого/оранжевого)
- Создание объема через цвет, а не только яркость
- Понимание того, как свет влияет на цвет объекта

## Инструкции

### Подготовка

1. **Прочитайте** `docs/pixel_art_fundamentals.md` (раздел 4: свет и тень, hue shift)
2. **Изучите** результаты Exercise 002 (ограниченная палитра)
3. **Откройте** `ui/test/TestButtonWithSquare.gd` для практики

### Шаги выполнения

1. **Выберите базовый цвет объекта** (например, золотистый для монеты, красный для яблока)

2. **Примените hue shift для теней**:
   - Тени: сдвиг базового цвета в сторону синего/фиолетового
   - Пример: золотистый (0.9, 0.75, 0.4) → темно-золотистый с синеватым оттенком (0.6, 0.5, 0.3)
   - Не просто затемнение, а изменение оттенка

3. **Примените hue shift для бликов**:
   - Блики: сдвиг базового цвета в сторону желтого/оранжевого
   - Пример: золотистый (0.9, 0.75, 0.4) → светло-золотистый с желтоватым оттенком (0.95, 0.8, 0.5)
   - Не просто осветление, а изменение оттенка

4. **Создайте объем**:
   - Используйте тени (hue shift в синий) для областей дальше от света
   - Используйте блики (hue shift в желтый) для областей ближе к свету
   - Источник света: сверху-слева (стандарт)

5. **Валидация**:
   - Проверьте, что тени имеют синеватый оттенок
   - Проверьте, что блики имеют желтоватый оттенок
   - Объект должен выглядеть объемным
   - Пройдите `docs/pixel_art_quality_checklist.md`
   - Цель: 70%+ пунктов пройдено

## Референсные примеры

### Что делает пример хорошим

- **Объем**: Объект выглядит объемным, не плоским
- **Естественность**: Освещение выглядит естественно
- **Гармония**: Цвета работают вместе, создают приятное визуальное впечатление
- **Читаемость**: Объект легко читается, форма понятна

### Принципы, которые применены

- **Hue shift для теней**: Тени имеют синеватый оттенок (сдвиг в сторону синего спектра)
- **Hue shift для бликов**: Блики имеют желтоватый оттенок (сдвиг в сторону желтого спектра)
- **Естественное освещение**: Свет влияет на цвет, а не только на яркость

### Как воспроизвести

1. Выберите базовый цвет объекта
2. Создайте тень: затемните базовый цвет И сдвиньте hue в сторону синего
3. Создайте блик: осветлите базовый цвет И сдвиньте hue в сторону желтого
4. Примените тени к областям дальше от источника света
5. Примените блики к областям ближе к источнику света

## Типичные ошибки

### 1. Простое затемнение/осветление без hue shift

**Симптомы**: Тени и блики выглядят плоскими, объект не объемный

**Причина**: Использование только изменения яркости без изменения оттенка

**Как исправить**: 
- Применяйте hue shift: тени в синий, блики в желтый
- Не просто затемняйте/осветляйте базовый цвет

### 2. Слишком сильный hue shift

**Симптомы**: Тени слишком синие, блики слишком желтые, объект выглядит неестественно

**Причина**: Преувеличенный сдвиг оттенка

**Как исправить**: 
- Используйте умеренный hue shift (небольшой сдвиг)
- Тени должны быть слегка синеватыми, не ярко-синими
- Блики должны быть слегка желтоватыми, не ярко-желтыми

### 3. Неправильное размещение теней и бликов

**Симптомы**: Объект выглядит неправильно освещенным, объем не читается

**Причина**: Неправильное понимание направления света

**Как исправить**: 
- Источник света: сверху-слева (стандарт)
- Тени: нижняя правая часть объекта
- Блики: верхняя левая часть объекта

### 4. Отсутствие плавных переходов

**Симптомы**: Резкие границы между тенями, бликами и основным цветом

**Причина**: Недостаточное количество промежуточных цветов или неправильное размещение

**Как исправить**: 
- Создайте плавные переходы между цветами
- Используйте промежуточные оттенки для плавности
- Размещайте пиксели так, чтобы переходы были естественными
